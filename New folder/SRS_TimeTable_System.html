1. Introduction

1.1 Purpose

This document provides a detailed Software Requirements Specification for the University Time Table Creation System, which automates conflict-free university timetable generation. It is intended for developers, project managers, testers, and stakeholders.

1.2 Scope

A web-based application for generating, managing, and viewing university timetables. Supports three user roles: Administrator, Teacher, and Student with role-specific dashboards. Uses a CSP solver with backtracking for conflict-free scheduling.

Out of Scope: Examination scheduling, fee management, attendance tracking, mobile native app.

2. Overall Description

2.1 Product Perspective

Self-contained web application replacing manual timetable creation. React.js front-end communicating with Node.js/Express.js back-end via RESTful APIs. Data persisted in JSON files.

2JSON file-based storage; no external database server required

3.9 Functional Requirements — Timetable Generation

REQ-IDRequirementPriority
FR-TT-01Admin can generate a timetable for a specific department and semester.High
FR-TT-02Uses CSP algorithm with backtracking for conflict-free scheduling.High
FR-TT-03aHard Constraint: No teacher assigned to two classes at the same time.High
FR-TT-03bHard Constraint: No classroom assigned to two classes at the same time.High
FR-TT-03cHard Constraint: No student group has two classes at the same time.High
FR-TT-03dHard Constraint: Lab sessions must not cross lunch break.High
FR-TT-03eHard Constraint: Room type must match session type.High
FR-TT-04aSoft Constraint: Spread course lectures across different days.Medium
FR-TT-04bSoft Constraint: Balance daily class load per student group.Medium
FR-TT-04cSoft Constraint: Penalize 3+ consecutive classes.Medium
FR-TT-04dSoft Constraint: Lectures prefer morning; labs prefer afternoon.Medium
FR-TT-04eSoft Constraint: Distribute teacher load evenly across days.Medium
FR-TT-04fSoft Constraint: Prefer smallest room that fits batch size.Medium
FR-TT-05Uses MRV heuristic to schedule most constrained sessions first.Medium
FR-TT-06Supports backtracking (up to 50 iterations) to resolve failures.Medium
FR-TT-07Reports errors for sessions that could not be placed.High
FR-TT-08Supports generating timetable for a specific batch.Medium
FR-TT-09Generate timetables for all semesters (1–8) of a department at once.Medium
FR-TT-10Delete all timetable entries for a specific department+semester.Medium
